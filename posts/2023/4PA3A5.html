Quasar 项目中的别名

发表于 2023-09-08 分类于 Develop ， Javascript ， Quasar 本文字数： 641 阅读时长 ≈ 1 分钟

使用 Quasar CLI 与 Vite 搭建 vue 项目后，没有经典的 vite.config.js 文件，相反，只有 quasar.config.js文件（更多详细信息：https://quasar.dev/quasar-cli-vite/quasar-config-file）。

在这种情况下，如何为 src 目录设置别名呢？

如果有使用 ElementUI Admin 的经验，可能会将 src 设置成别名 @，然而在 Quasar 中，官方建议不将将 @ 当作 src 的别名，因为 npm 包中，组织包是以 @ 开头，会发生冲突。

在所有项目中，最好都不要使用 @ 作为 src 别名

系统别名

Quasar 为我们提供了配置好的别名，可以直接使用，它们分别是：

Alias Resolves to
src /src
app /
components /src/components
layouts /src/layouts
pages /src/pages
assets /src/assets
boot /src/boot

自定义别名

如果上述别名无法满足日常使用，还可以通过这个教程添加自定义别名：Adding Webpack aliases
