会话 Session

Session 是 MongoDB 3.6 之后引入的概念，在以前的版本中，Mongod 进程中的每一个请求会创建一个上下文（OperationContext），可以理解为一个单行事务，这个单行事务中对于数据、索引、oplog 的修改都是原子性的。

MongoDB 3.6 之后的 Session 本质上也是一个上下文，在这个 Session 会话中多个请求共享一个上下文，为多文档事务实现提供了基础。
